[题目传送门。](https://www.luogu.com.cn/problem/P6076)

这题是计数题，很容易想到容斥。

先考虑全集 $U$ 的大小，对于每一个格子，它可以选择涂 $C$ 中颜色之一或者不涂，所以：

$$
|U| = (C+1)^{nm}
$$

由于这道题要求满足所有约束条件，所以我们将题目中的三个约束条件转化为“坏性质”条件集合 $A,B,D$，具体地：
- $A_i$ 表示第 $i$ 行没有任何格子被涂色。
- $B_j$ 表示第 $j$ 行没有任何格子被涂色。
- $D_k$ 表示第 $k$ 中颜色在整个棋盘上一次都没有出现。

设答案为 $ans$，则：

$$
ans = \sum_{I \in \{1,2,\dots,n\}} \sum_{J \in \{1,2,\dots,m\}} \sum_{K \in \{1,2,\dots,C\}} (-1)^{|I|+|J|+|K|} \left|\bigcap_{i \in I} A_i \cap \bigcap_{j \in J} B_j \cap \bigcap_{k \in K} D_k\right|
$$
很容易发现我们可以消除掉前两条“坏性质”，因为 $I$ 中的行中每个格子都不能染色，$J$ 中的列中每个格子都不能染色，所以令 $|I| = a,|J| = b$，$cnt$ 表示剩余可涂色的格子，根据两个集合的容斥原理则有：

$$
\begin{align*}
cnt &= nm-(am+bn-ab) \\
&= nm-am-bn+ab \\
&= (n-a) \times (m-b)
\end{align*}
$$

感性理解：删掉了 $a$ 行，去除了 $b$ 列，只剩 $(n-a) \times (m-b)$ 这个小矩阵里的所有格子是可涂色的了。

接下来还有最后一个约束，令 $|K| = x$，很容易转化成我们只有 $C-x$ 个颜色可以涂，同时我们还可以选择不涂色，也就是说我们对于每个可涂色的格子有 $(C-x+1)$ 个方案，所以：

$$
\left|\bigcap_{i \in I} A_i \cap \bigcap_{j \in J} B_j \cap \bigcap_{k \in K} D_k\right| = (C-x+1)^{(n-a)(m-b)}
$$
很容易发现 $(C-x+1)^{(n-a)(m-b)}$ 只和 $|I|,|J|,|K|$ 有关，和 $I,J,K$ 的具体内容无关，所以我们可以先枚举 $I,J,K$ 的长度 $a,b,x$，然后将 $I,J,K$ 的选择方案和对应的贡献分离，得到：

$$
ans = \sum_{a = 0}^n \sum_{b = 0}^m \sum_{x = 0}^C (-1)^{a+b+x} \binom{n}{a} \binom{m}{b} \binom{C}{x} (C-x+1)^{(n-a)(m-b)}
$$

不过这么做的时间复杂度是 $O(nmC \log nm)$ 的，极限情况下大约循环 $10^9$ 次，不足以通过。

加个预处理幂次？那你还是太天真了，这样时间复杂度虽然降到了 $O(nmC)$，但是我们忽略了常数 $2$，于是极限情况下大约循环 $1.28 \times 10^8$ 次，这不是能通过吗？其实我们还忽略了大量取模巨大的开销，这样一来就无法通过了。

这题的时间复杂度的瓶颈有两个，预处理幂次部分和计算答案部分，计算答案部分的时间复杂度肯定是优化不了了，但是预处理幂次的时间复杂度可以优化啊！将答案进行转化：

$$
\begin{align*}
ans = \sum_{x = 0}^C (-1)^x \binom{C}{x} S(k),\ \ \ \ \ S(k) &= \sum_{a = 0}^n \sum_{b = 0}^m (-1)^{a+b} \binom{n}{a} \binom{m}{b} (C-x+1)^{(n-a)(m-b)} \\
&= (-1)^{n+m}\sum_{p = 0}^n \sum_{q = 0}^m (-1)^{p+q} \binom{n}{p} \binom{m}{q} (C-x+1)^{pq},\ \ \ \ \ p = n-a,q = m-b \\
&= (-1)^{n+m}\sum_{p = 0}^n \sum_{q = 0}^m (-1)^{p+q} \binom{n}{p} \binom{m}{q} ((C-x+1)^p)^q
\end{align*}
$$

很容易发现我们只需要预处理所有的 $(C-x+1)^p$ 即可，$q$ 次幂是可以在循环中计算的。

于是预处理幂次的时间复杂度就从 $O(nmC)$ 变成了 $O(nC)$。

于是就可以轻松通过了，至于为什么呢？因为预处理幂次也有取模运算，这种方法使得取模次数大大减小，开销也是大大减小（不要小看取模的开销），自然就通过得很轻松。

---
代码：
```cpp
#include<bits/stdc++.h>
using namespace std;
const int mod = 1e9+7;
int jie[405];
int inv[405];
int mi[405][160005];
int fast_pow(int x,int y)
{
	int ans = 1;
	while(y)
	{
		if(y&1)
		{
			ans = 1ll*ans*x%mod;
		}
		x = 1ll*x*x%mod;
		y>>=1;
	}
	return ans;
}
int C(int n,int m)
{
	return 1ll*jie[n]*inv[m]%mod*inv[n-m]%mod;
}
int main()
{
	ios_base::sync_with_stdio(0);
	cin.tie(0);
	cout.tie(0);
	int n,m,c;
	cin >> n >> m >> c;
	int s = max(max(n,m),c);
	jie[0] = 1;
	for(int i = 1;i<=s;++i)
	{
		jie[i] = 1ll*jie[i-1]*i%mod;
	}
	inv[s] = fast_pow(jie[s],mod-2);
	for(int i = s-1;i>=0;--i)
	{
		inv[i] = 1ll*inv[i+1]*(i+1)%mod;
	}
	int ss = c+1;
	for(int i = 1;i<=ss;++i)
	{
		mi[i][0] = 1;
		for(int j = 1;j<=n;++j)
		{
			mi[i][j] = 1ll*mi[i][j-1]*i%mod;
		}
	}
	int sum = 0;
	for(int k = 0;k<=c;++k)
	{
		int cnt = 0;
		for(int i = 0;i<=n;++i)
		{
			int ans = 1,w = mi[c-k+1][i];
			for(int j = 0;j<=m;++j)
			{
				int q = 1ll*C(n,i)*C(m,j)%mod;
				if(i&&j)
				{
					q = 1ll*q*ans%mod;
				}
				if(i+j&1)
				{
					cnt-=q;
					if(cnt<0)
					{
						cnt+=mod;
					}
				}
				else
				{
					cnt+=q;
					if(cnt>=mod)
					{
						cnt-=mod;
					}
				}
				if(w!=1)
				{
					ans = 1ll*ans*w%mod;
				}
			}
		}
		cnt = 1ll*cnt*C(c,k)%mod;
		if(n+m+k&1)
		{
			sum-=cnt;
			if(sum<0)
			{
				sum+=mod;
			}
		}
		else
		{
			sum+=cnt;
			if(sum>=mod)
			{
				sum-=mod;
			}
		}
	}
	cout << sum;
	return 0;
}
```
